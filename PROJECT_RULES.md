# PROJECT_RULES.md

Version: 2.0
Last updated: 2026-07-09

## Stack

- Python 3.14
- Flask 3.1
- SQLite
- HTML5, CSS3
- Vanilla JavaScript

## Architecture

- Использовать Flask Blueprint.
- Не создавать новые Blueprint без необходимости.
- Бизнес-логика находится в `services/`.
- Маршруты должны быть максимально тонкими (thin routes).
- Новая функциональность добавляется через новый Blueprint.

## Database

- SQLite.
- Все SQL-запросы параметризованные.
- Без конкатенации SQL.
- Для денежных значений использовать Decimal.
- float запрещён для финансовых расчётов.

## Frontend

- Только Vanilla JS.
- Не использовать jQuery.
- CSS без inline-стилей.
- Не использовать глобальные JS-переменные.

## HTML

- Jinja2.
- Автоэкранирование не отключать.

## Code Style

- PEP8.
- Type hints обязательны для новых функций.
- Комментарии только там, где логика неочевидна.
- Нет комментариев-пояснений вида "этот код делает X".

## Logging

- Использовать logging.
- print() запрещён.

## Dependencies

- Не добавлять новые библиотеки без согласования с Architect.

## Security

- Не хранить секреты в коде.
- Использовать .env.
- Все пользовательские данные валидировать.

## Tests

- Если меняется бизнес-логика — обновить тесты.
- QA и Fixer работают ТОЛЬКО с временной копией БД.
- Запрещено напрямую изменять production-файл `finance.db`.
- Для тестов всегда создавать копию в `tests/temp/`.
- После завершения тестов удалять временную БД.
- Никогда не полагаться на ручной backup/restore `finance.db`.
- Для создания временной БД использовать шаблон:

```python
import os, shutil
os.makedirs('tests/temp', exist_ok=True)
tmp_db = os.path.join('tests/temp', 'finance_test.db')
shutil.copy2('finance.db', tmp_db)
try:
    import database
    database.DB_PATH = tmp_db
    # ... тесты ...
finally:
    if os.path.exists(tmp_db):
        os.remove(tmp_db)
```

## Performance

- Избегать повторных SQL-запросов.
- Не выполнять запросы внутри циклов (N+1).

## AI Rules

Перед изменением кода:

1. Прочитать PROJECT_RULES.md.
2. Изучить соседние файлы.
3. Не менять стиль проекта.
4. Не выполнять рефакторинг без запроса.
5. Не переименовывать существующие функции без необходимости.
6. Не менять публичные API без согласования.
7. Предпочитать минимальные изменения.

## Existing Code Priority

Если существующая архитектура отличается от рекомендуемой — следовать существующей архитектуре проекта. Не переписывать проект ради «лучших практик».

## Temporary Files

- Не создавать временные скрипты и файлы в корне проекта.
- Для временных инструментов использовать `tests/temp/`.
- После завершения задачи удалять все созданные временные файлы.
- Файлы вида `analyze*.py`, `check_*.py`, `dump_*.py`, `fix_*.py`, `revert_*.py`, `verify_*.py` и подобные одноразовые скрипты не должны оставаться в проекте.

## Financial Rules

- Для денежных значений использовать Decimal.
- Не использовать float.
- Все расчёты должны быть детерминированными.
- Округление выполнять только при отображении, а не при хранении.
- Валюту хранить отдельно от суммы.

## Definition of Done

Задача считается выполненной только если:

- реализована полностью;
- не нарушены правила проекта;
- Reviewer выдал «Принять»;
- QA выдал «Пройдено»;
- нет критичных замечаний;
- пользователь получил итоговый отчёт.
